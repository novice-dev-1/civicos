STEP 1.2 — Create backend/app/models.py
  Four SQLAlchemy 2.0 models using Mapped[] typing:

  class Incident(Base):
    id               : Mapped[uuid.UUID]   PRIMARY KEY, default gen_random_uuid()
    description      : Mapped[str]         TEXT NOT NULL
    type             : Mapped[str | None]   VARCHAR(20)
    severity         : Mapped[str | None]   VARCHAR(20)
    victims          : Mapped[int]         DEFAULT 0
    specialty_required: Mapped[str | None] VARCHAR(50)
    lat              : Mapped[float | None]
    lng              : Mapped[float | None]
    status           : Mapped[str]         DEFAULT 'PENDING'
    priority_score   : Mapped[float | None]
    created_at       : Mapped[datetime]    TIMESTAMPTZ DEFAULT NOW()
    resolved_at      : Mapped[datetime | None]
    time_to_care_min : Mapped[float | None]

  class Resource(Base):
    id               : Mapped[uuid.UUID]   PRIMARY KEY
    type             : Mapped[str]         NOT NULL (ambulance|hospital|police)
    name             : Mapped[str]         NOT NULL
    lat              : Mapped[float]       NOT NULL
    lng              : Mapped[float]       NOT NULL
    capacity_total   : Mapped[int]         DEFAULT 1
    capacity_used    : Mapped[int]         DEFAULT 0
    specialty        : Mapped[str | None]  (hospitals only)
    equipment        : Mapped[list[str] | None]  ARRAY TEXT
    jurisdiction     : Mapped[str | None]  (police only)
    status           : Mapped[str]         DEFAULT 'AVAILABLE'

  class Assignment(Base):
    id               : Mapped[uuid.UUID]   PRIMARY KEY
    incident_id      : Mapped[uuid.UUID]   FK → incidents.id ON DELETE CASCADE
    resource_id      : Mapped[uuid.UUID]   FK → resources.id
    role             : Mapped[str | None]  VARCHAR(50)
    eta_min          : Mapped[float | None]
    distance_km      : Mapped[float | None]
    assigned_at      : Mapped[datetime]    DEFAULT NOW()

  class AuditLog(Base):
    id               : Mapped[uuid.UUID]   PRIMARY KEY
    incident_id      : Mapped[uuid.UUID]   FK → incidents.id ON DELETE CASCADE
    agent_name       : Mapped[str | None]  VARCHAR(50)
    step             : Mapped[int | None]
    thought          : Mapped[str | None]  TEXT
    action           : Mapped[str | None]  TEXT
    alternatives     : Mapped[dict | None] JSONB
    logged_at        : Mapped[datetime]    DEFAULT NOW()

  Production note: Add __table_args__ = {"schema": "public"} to each model
  for explicit Neon schema targeting.

STEP 4.5 — AmbulanceAgent (algorithmic)
  File: backend/app/agents/ambulance.py
  Score formula: 0.5*distance_norm + 0.3*equipment_match + 0.2*capacity_free
    distance_norm = 1 - (distance / max_distance_in_set)
    equipment_match = fraction of required equipment present
    capacity_free = (capacity_total - capacity_used) / capacity_total
  Return top 3 candidates as ambulance_candidates in state
  Mark each with score, rejection_reason if not top pick
  Write audit_log row.

STEP 4.6 — HospitalAgent (algorithmic)
  File: backend/app/agents/hospital.py
  Score formula: 0.4*specialty_match + 0.3*icu_available + 0.3*distance_norm
    specialty_match = 1.0 if exact match, 0.5 if general, 0.0 if mismatch
    icu_available = (capacity_total - capacity_used) / capacity_total
  Return top 3 candidates as hospital_candidates in state
  Write audit_log row.

STEP 4.7 — PoliceAgent (algorithmic)
  File: backend/app/agents/police.py
  Score formula: 0.5*jurisdiction_match + 0.5*distance_norm
  Return top 3 candidates as police_candidates in state
  Write audit_log row.

STEP 4.8 — TrafficAgent (algorithmic)
  File: backend/app/agents/traffic.py
  ETA calculation: distance_km / 30 * 60  (30 km/h avg Delhi speed)
  Corridor logic:
    if eta > 8 min AND severity in [HIGH, CRITICAL]:
      corridor_plan = {
        "route": "suggest fastest path description",
        "police_escort_needed": True,
        "recommended_police_unit": top police candidate
      }
  Write audit_log row.

STEP 4.9 — ForecastAgent (algorithmic)
  File: backend/app/agents/forecast.py
  Input: rolling incident history (last 60 min) from DB
  Logic:
    active_count = incidents with status PENDING or ASSIGNED
    available_ambulances = count resources WHERE type=ambulance AND status=AVAILABLE
    if active_count >= available_ambulances:
      forecast_warning = "Resource shortage predicted — pre-position standby units"
    else:
      forecast_warning = None
  Write audit_log row.

================================================================================
  SECTION 2 — PRODUCTION HARDENING (cross-cutting, apply throughout)
================================================================================

These must be addressed at each phase, not saved for the end.

H1 — Environment variable validation at startup
  In app/config.py, Settings class with pydantic-settings:
    @validator('DATABASE_URL') → verify it starts with postgresql+asyncpg://
    @validator('GEMINI_API_KEY') → verify non-empty
  If any required var is missing, fail fast with a clear error message.
  Never let the app start with empty secrets.

H2 — Database connection pooling
  In app/db.py, set pool_size=5, max_overflow=10, pool_timeout=30
  For Neon serverless, use NullPool to avoid connection pinning:
    from sqlalchemy.pool import NullPool
    engine = create_async_engine(DATABASE_URL, poolclass=NullPool)
  This is critical for Neon — do not use the default pool with serverless.

H3 — CORS configuration (production)
  In app/main.py, CORSMiddleware:
    Development: allow_origins=["http://localhost:5173", "http://127.0.0.1:5173"]
    Production:  allow_origins=[settings.FRONTEND_URL]  (from .env)
    Never use allow_origins=["*"] in production.

H4 — Proper HTTP status codes
  201 Created → POST /incidents (not 200)
  404 Not Found → GET /incidents/{id} when id doesn't exist
  422 Unprocessable → invalid input (Pydantic handles this automatically)
  500 Internal Server Error → unhandled exceptions (with error logging)
  Add global exception handler in main.py for 500s.

H5 — Input validation and sanitization
  Incident description: max 2000 characters
  Coordinates: lat in [-90, 90], lng in [-180, 180]
  Validate in Pydantic schemas with Field(max_length=2000) and validators.
  Never pass raw user input directly to Gemini prompts without escaping.

H6 — Structured logging
  Use Python's logging module (not print statements)
  Log format: ISO timestamp, level, module, message
  In production (Render), logs go to stdout → captured by Render dashboard
  Log: request method+path+status+duration for all requests
  Do NOT log: raw Gemini responses, database passwords, JWT tokens

H7 — Graceful shutdown
  In app/main.py, use @asynccontextmanager lifespan:
    On startup: verify DB connection, log "JIRACHI started"
    On shutdown: close DB pool, drain broadcaster queues, log "JIRACHI stopping"

H8 — Rate limiting (basic, for demo safety)
  Add slowapi (pip install slowapi) to backend
  Rate limit POST /incidents: 10/minute per IP
  Rate limit POST /admin/reset: 1/minute (nuclear button)
  This prevents accidental DoS during demo.

H9 — SSE reconnection (frontend)
  EventSource reconnects automatically by spec.
  But set: eventSource.onerror → exponential backoff (1s, 2s, 4s, max 10s)
  After 5 failed reconnects, fall back to 2s polling.
  Show a subtle "Reconnecting..." indicator in AgentFeed.

H10 — Render cold start mitigation
  Render free tier sleeps after 15 min of inactivity.
  Solution A (preferred): Use UptimeRobot (free) to ping /health every 10 min.
  Solution B: Run demo from local backend during actual demo recording.
  Solution C: Set DEMO_MODE=true on Render to make cold-start responses instant.
  For hackathon demo: always have local backend running as backup.

H11 — Neon connection limits
  Neon free tier: max 3 concurrent connections.
  Use NullPool (see H2). Open connection, run query, close immediately.
  If you hit "too many connections": check for unclosed sessions in tests.

H12 — Audit log size cap
  Cap at 200 entries per incident (per the plan's risk checklist).
  In AuditLog writes: if count > 200, skip with a warning log.
  Add a cleanup job concept (not needed for hackathon, note for production).
